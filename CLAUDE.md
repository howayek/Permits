## Auth Setup
- **Provider:** Supabase Auth (email/password). Clerk is NOT used anywhere.
- **Email confirmation:** ON (users must verify email before first login)
- **Trigger:** `on_auth_user_created` on `auth.users` → inserts into `user_roles` with role from `raw_user_meta_data->>'role'`, defaults to CITIZEN
- **Profiles table:** Does NOT exist (was dropped by friend). No trigger references it.
- **Auth architecture:** Roles fetched in AuthProvider and stored in context. Primary source: `user_roles` table query. Fallback: JWT `user_metadata.role` (for when DB query returns empty). `useAuth()` is a pure context consumer — no local state, no race conditions.
- **Context exposes:** `user`, `session`, `roles` (string[]), `isDeveloper`, `isGovernment`, `loading`
- **`has_role(p_role app_role)`** — SECURITY DEFINER Postgres function that checks `user_roles` for `auth.uid()`. Used by all gov RLS policies.

## UI Architecture
- **Toaster:** `<Toaster />` from shadcn/ui is mounted in `main.tsx` for toast notifications
- **Header:** Role-aware — citizens see Apply/My Permits; government sees Review Queue/Applications
- **Dashboard:** Three views — citizen (apply + my permits cards), government (review queue + database cards), developer (DeveloperDashboard component)
- **OwnedPermits categorization:** Uses both `decisions.decision` value AND `applications.status` to determine phase. Falls back to app status when decision row isn't visible.
